## 📋 什么是"Agent 模式"

不是简单问答，而是：

```
用户给目标 → Agent 自己拆解 → 自己选择工具 → 自己执行 → 自己判断完成度 → 反馈结果

例：
"帮我整理桌面" 
→ 识别桌面路径
→ 扫描文件类型
→ 按日期/类型分类
→ 移动文件
→ 生成整理报告
→ 告知用户完成
```

**关键特征**:
- 🧠 **自主规划**: 不依赖用户一步步指导
- 🛠️ **工具选择**: 知道什么时候用什么工具
- 🔄 **自我纠错**: 失败了自己重试或换方案
- 💾 **记忆积累**: 从经验中学习，越用越聪明

**关键文件**: `AGENTS.md`（已存在，需扩展）

```markdown
## Agent 模式工作流程

当用户提出请求时：

1. **理解意图**: 分析用户想要什么结果
2. **拆解任务**: 把大目标拆成小步骤
3. **选择工具**: 根据步骤选择合适 Skill
4. **执行并监控**: 调用工具，观察结果
5. **自我纠错**: 失败时重试或换方案
6. **汇总输出**: 整理结果，反馈用户

**示例**:
用户: "整理下载文件夹"
→ 拆解: [扫描文件] → [分类] → [移动] → [报告]
→ 工具: file_manager + system_monitor
→ 执行: 自动完成
```
