# 2026-02-20 - WeChat Automation Research - 项目已封存

## 项目状态: 🔴 已封存

### 根本原因
微信 4.x 使用**自定义渲染引擎** `MMUIRenderSubWindowHW`，不实现标准 UIAutomation 接口，导致：
- wxauto、wxautold 无法访问内部控件
- pywinauto、uiautomation 全部失效
- 监听功能无法实现

### 最终结论
| 功能 | 状态 | 原因 |
|------|------|------|
| **发送消息** | ✅ 可用 | pyautogui 模拟点击 |
| **监听消息** | ❌ 不可用 | 自定义渲染引擎限制 |
| **读取消息** | ❌ 不可用 | 自定义渲染引擎限制 |
| **发送图片** | ✅ 可用 | pyautogui 模拟点击 |

### 微信窗口结构 (问题根源)
```
WeChat Window (Qt51514QWindowIcon)
  └── MMUIRenderSubWindowHW  ← 隐藏所有内部UI控件
       ├── 聊天列表 (不可访问)
       ├── 输入框 (不可访问)
       └── 发送按钮 (不可访问)
```
